# AI-Chat Development Tasks

> 适用当前仓库：`Vue 3 + Vite + TypeScript + FastAPI + PostgreSQL + JWT + LangChain`
>
> 当前状态：注册登录、JWT 认证、导航守卫、聊天页面、会话管理、聊天记录、PostgreSQL 接入已基本完成。
>
> 本文档目标：把后续开发拆成可连续执行的阶段任务，优先完成本地上传、PDF 入库、RAG，再做图片多模态与稳定性增强。

---

## 使用方式

推荐按下面方式使用，不要一次性让 AI 实现全部阶段。

1. 先让 AI 阅读 `CLAUDE.md` 和本文件 `tasks.md`
2. 每次只选择一个阶段执行，例如 `P0` 或 `P1`
3. 把对应阶段的“执行提示词”发给 AI
4. 要求 AI 先输出实施方案，再开始编码
5. 编码完成后，要求 AI 输出：
   - 修改了哪些文件
   - 为什么这样改
   - 做了哪些验证
   - 当前阶段还有哪些未完成项
6. 本地验收通过后，再执行下一阶段
7. 每完成一个阶段，单独提交一次 Git

建议固定指令模板：

```text
请先阅读根目录 CLAUDE.md 和 tasks.md。
现在只执行 tasks.md 中的 [P0/P1/P2/P3/P4/P5]，不要提前实现下一个阶段。
要求：
1. 先基于现有代码给出实施计划
2. 只修改和当前阶段相关的文件
3. 前后端联动时同步补齐类型、schema、接口
4. 完成后列出修改文件、验证结果、剩余风险
5. 最后给出建议的 git commit message
```

## P3 RAG 检索问答

### 目标

把知识库真正接入聊天链路，让问题先检索文档片段，再生成带引用的回答。

### 前端任务

- [ ] 聊天页增加知识库问答开关或模式选择
- [ ] 展示引用来源
- [ ] 展示命中文档名、页码、片段摘要
- [ ] 支持无命中时的普通聊天回退提示

### 后端任务

- [ ] 确定向量方案，优先 `pgvector`
- [ ] 增加 Embedding 生成流程
- [ ] 为 chunk 建立向量索引
- [ ] 新增 retriever service
- [ ] 在聊天流程中接入检索增强上下文
- [ ] 回答结果中返回引用来源

### 建议接口演进

- 方案 A：扩展现有 `/chat` 接口，增加 `mode` 或 `use_rag`
- 方案 B：新增独立 RAG 聊天接口

优先建议：先走方案 A，减少前后端平行体系。

### 验收标准

- [ ] 文档 chunk 可完成 embedding
- [ ] 用户问题可召回相关片段
- [ ] 模型回答可附带引用来源
- [ ] 无命中时能回退到普通聊天
- [ ] 不串入其他用户文档数据

### 执行提示词

```text
请先阅读根目录 CLAUDE.md 和 tasks.md。
现在只执行 P3 RAG 检索问答，不要实现图片多模态。

目标：

2. 新增 retriever，根据问题召回最相关 chunk
3. 将召回结果拼进 LLM 上下文，生成带引用的答案
4. 前端展示文档名、页码、片段摘要等引用信息
5. 无命中时自动回退普通聊天

要求：
- 检索和生成逻辑解耦
- 保持现有普通聊天功能可用
```
